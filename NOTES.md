# PINN Project — Notes

## Що це

Physics-Informed Neural Network для 1D рівняння теплопровідності з рухомим пальником.

Задача: є металева труба довжиною l, по ній рухається пальник зі швидкістю v.
Знайти розподіл температури ΔT(x, t) для будь-якого матеріалу та швидкості.

Рівняння:
```
∂ΔT/∂t = α · ∂²ΔT/∂x² + i_eff · δ(x − x₀ − v·t)
```
де α — теплодифузійність матеріалу, i_eff — потужність пальника.

## Архітектура мережі

**Вхід:** (x_norm, t_norm, α_n, l_n, i_eff_n, x₀_n, v_n) — 7 чисел
**Вихід:** ΔT — підйом температури в точці x в момент t

**Pipeline:**
```
(x, t) → FourierFeatures(σ) → 128 ознак
                                        ┐
(α, l, i_eff, x₀, v) ─────────────────→ cat → [133] → MLP(4×256, tanh) → × t_norm → ΔT
```

`× t_norm` — hard IC: гарантує ΔT(x,0)=0 математично, без штрафного терму.

Параметри: ~232k ваг.

---

## Три ключові покращення (вже реалізовані)

### 1. Hard Initial Condition
`output = t_norm * MLP(features)` → при t=0 вихід завжди 0.
До цього: мережа порушувала IC, що "псувало" всю еволюцію в часі.

### 2. Adaptive Fourier Sigma (curriculum)
`σ` росте з 1.0 до 3.5 лінійно протягом навчання.
Спочатку мережа вчить плавні глобальні патерни, поступово деталізує гострий пік пальника.

### 3. Causal PDE Loss (Wang et al., JMLR 2024)
Ділимо час на 10 вікон. Вікно k отримує вагу:
```
w_k = exp(−ε · Σ_{j<k} L_j)
```
Поки рання динаміка (t малий) погана — пізні вікна майже вимкнені.
Мережа мусить вивчити початок перш ніж переходити до продовження.

**Що видалено:** L-BFGS фаза (підвищувала помилку з 186% → 289%).

## Архітектура: PirateNets (вже реалізовано)

Wang et al., JMLR 2024. Кожен блок:
```
U = tanh(W_u · features),  V = tanh(W_v · features)   ← shared encoders (1 раз за forward pass)

h = tanh(W1 · x)
h = h⊙U + (1−h)⊙V        ← gate 1
h = tanh(W2 · h)
h = h⊙U + (1−h)⊙V        ← gate 2
h = tanh(W3 · h)
out = α·h + (1−α)·x       ← learnable residual gate (α init=0, unconstrained)
```
695 300 параметрів. Поточний результат: ~68% на кроці 60k (краще за baseline 76%).

## Наступні покращення (дослідження літератури, 2025)

### 1. FP64 — double precision (реалізовано, вимкнено за замовчуванням)

**Джерело:** "FP64 is All You Need", NeurIPS 2025 (arXiv:2505.10949)

**Проблема яку вирішує:** У FP32 Adam може застрягти в "stall phase" — loss зупиняється
але помилка залишається великою через accumulation помилок округлення у градієнтах.

**Реалізація:** `use_fp64: true` в config.yaml → `model.double()` + cast батчів.

**Осторога:** Стаття фокусується на L-BFGS + FP64. Для Adam ефект менший (~10-20%),
але коштує лише зміна одного параметра.

---

### 2. RAD — Residual-based Adaptive Distribution (реалізовано, вимкнено за замовчуванням)

**Джерело:** "A comprehensive study of non-adaptive and residual-based adaptive sampling
for PINNs", arXiv:2207.10289. Convergence proof: arXiv:2504.00910 (2025).

**Проблема яку вирішує:** Рандомне рівномірне семплювання марнує точки там де residual
малий. Для рухомого пальника — він переміщується в просторі, і residual сконцентрований
вузько навколо x_b(t). RAD автоматично ставить більше точок де потрібно.

**Ідея:**
```
1. Семплуємо pool_factor × n_pde = 40 000 кандидатів рівномірно
2. Обчислюємо |residual| кінцевими різницями (4 forward pass, no_grad)
3. Семплуємо n_pde = 8 192 точки з ймовірністю ∝ |residual|
4. Повторюємо кожні rad_update_every = 50 кроків (кешуємо між оновленнями)
```

**Чому кінцеві різниці а не autograd:**
Для RAD нам потрібні тільки значення residual, а не їхні градієнти по вагах.
FD (4 forward passes) дає ~40× менше пам'яті ніж autograd + create_graph=True
при такому великому пулі.

---

### 3. ProPINN (ідея для майбутнього / пояснення в проекті)

**Джерело:** "ProPINN: Demystifying Propagation Failures in Physics-Informed Neural
Networks", arXiv:2502.00803, лютий 2025.

**Ключова ідея:** Доводить що провали PINN на часових задачах виникають через
низьку кореляцію градієнтів між сусідніми точками по часу (через NTK):

```
K(t₁, t₂) = ⟨∇θ f(t₁), ∇θ f(t₂)⟩ ≈ 0   ← мережа не може "передати" знання
```

Якщо оновлення ваг для t=0.1 ламає t=0.15 (антикорельовані градієнти) — мережа
не може поширювати рішення вперед по часу. Causal loss пом'якшує симптом,
ProPINN вирішує причину через sampling по кореляції градієнтів.

**Складність імплементації:** Висока (потрібен NTK або його апроксимація).
Рекомендується описати в проекті як "root cause analysis" а не імплементувати.
Покращення: +46% відносно трансформерних PINN на конвекційних задачах.

## Реалістичний прогноз помилки

```
Поточний стан (PirateNets, 120k кроків): ~40-60%  (очікуємо після завершення)
+ RAD:                                    → ~25-40%
+ FP64:                                   → ~20-35%
```

5% без аналітичної формули для параметричного PINN (5 параметрів одночасно)
потребує або набагато більшої мережі, або temporal decomposition.
